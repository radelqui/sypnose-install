# Iron Laws — Sypnose (Boris + Karpathy + Superpowers)

## 13 Laws (violate = work rejected)

### Sypnose Core
1. **Sin boris_start_task no hay tarea** — invisible work = lost work
2. **Sin OK del usuario no se envia** — never auto-deploy without approval
3. **Sin verificacion no hay resultado** — hook blocks commit without evidence
4. **Sin §11 el agente ejecuta ciego** — agent MUST improve bad instructions
5. **Sin MEJORA CONTINUA no aprendes** — 3 dimensions feedback every task
6. **Sin capture-pane no se despacha** — check agent state before sending
7. **NUNCA 2 prompts al mismo agente** — while working, NO interruptions
8. **NUNCA dejar agentes idle** — if there's work, dispatch it

### Boris Cherny 2026
9. **Plan before code** — pour energy into plan, 1-shot implementation
10. **Delegation > pair programming** — goal + constraints + criteria upfront

### Karpathy 4 Principles
11. **Think before coding** — don't assume; state assumptions explicitly
12. **Simplicity first** — minimum code that solves the problem
13. **Surgical changes** — fix ONLY what's asked, don't improve adjacent code

## Anti-Patterns (from 24 failure memories)
- "Should work" without running verification
- Expressing satisfaction before evidence
- Trusting agent success reports without independent check
- Sending multiple prompts during active work
- Estimating time/cost in plans (creates bias)
- Using "TODO" or "TBD" in plans (lazy planning)
