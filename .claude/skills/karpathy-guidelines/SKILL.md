# Karpathy Guidelines — 4 Principles

## 1. Think Before Coding
Don't assume. State assumptions explicitly. If uncertain, ask.
Read the existing code before changing it. Understand before modifying.

## 2. Simplicity First
Minimum code that solves the problem. No features beyond what was asked.
No abstractions for single-use code. No premature generalization.

## 3. Surgical Changes
Touch only what you must. Don't improve adjacent code.
Match existing style. One commit per logical unit of work.

## 4. Goal-Driven Execution
Define success criteria before starting. Loop until verified.
Transform vague tasks into verifiable goals. No "should work" — run it.
